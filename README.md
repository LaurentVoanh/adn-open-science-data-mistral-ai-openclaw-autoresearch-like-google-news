Vous êtes chercheur en biologie. Chaque matin, vous ouvrez votre ordinateur et vous découvrez :

## ❓ Mais concrètement, c'est quoi ?

C'est un **logiciel intelligent** qui :

1. **Lit l'actualité scientifique** (comme vous le feriez sur Google Actualités)
2. **Comprend ce qu'il lit** (il repère les gènes, les maladies, les méthodes)
3. **Va chercher les données** dans les grandes bases scientifiques mondiales
4. **Écrit un rapport de recherche** sourcé, clair, utile
5. **Apprend de ses erreurs** pour devenir meilleur à chaque cycle

**Tout ça automatiquement. Sans que vous ayez à cliquer, taper ou configurer.**

## 🔄 Comment ça marche ? (Étape par étape, en langage humain)

### 🌅 Étape 1 : L'agent "se réveille" (toutes les 3 minutes)

> *Comme vous qui ouvrez votre flux d'actualités le matin.*

- Il consulte **Google News** avec des filtres scientifiques
- Il cherche des mots comme : `gène`, `protéine`, `CRISPR`, `cancer`, `essai clinique`
- Il ignore les sujets non-scientifiques : `smartphone`, `promotion`, `gaming`
- Il sélectionne **le sujet le plus pertinent** du moment

📰 *Exemple trouvé* :  
> *"Nouvelle thérapie CRISPR pour la drépanocytose : résultats prometteurs"*

---

### 🧠 Étape 2 : L'agent "comprend" ce qu'il a lu

> *Comme vous qui lisez un titre et identifiez les concepts clés.*

- Il utilise l'IA pour extraire automatiquement :
  - 🧬 **Gènes** : TP53, BRCA1, HBB...
  - ⚠️ **Variants** : R175H, p.Arg175His...
  - 🦠 **Maladies** : drépanocytose, cancer, Alzheimer...
  - 🔬 **Méthodes** : CRISPR, RNA-seq, thérapie génique...

✅ **Vous ne saisissez rien.** L'agent devine tout seul.

---

### 🔍 Étape 3 : L'agent "enquête" dans les bases scientifiques

> *Comme vous qui iriez chercher des articles sur PubMed, mais en 10 secondes.*

Il interroge **5 bases de données ouvertes** :

| Base | Ce qu'elle contient | Ce que l'agent demande |
|------|-------------------|----------------------|
| **PubMed** | Articles de recherche médicale | *"Trouve-moi des articles sur CRISPR + drépanocytose"* |
| **UniProt** | Fonctions des protéines | *"À quoi sert la protéine HBB ?"* |
| **Ensembl** | Localisation des gènes dans l'ADN | *"Où se trouve le gène HBB dans le génome ?"* |
| **ClinVar** | Mutations et leur impact clinique | *"La mutation R175H est-elle dangereuse ?"* |
| **GitHub** | Outils open-source de recherche | *"Existe-t-il un outil PHP pour analyser ce variant ?"* |

🔁 Il pose ces questions **en parallèle**, pas l'une après l'autre.

---

### ✍️ Étape 4 : L'agent "rédige" un rapport scientifique

> *Comme un post-doc qui synthétiserait ses recherches pour vous.*

Il utilise l'IA pour écrire un rapport structuré :

```
📋 TITRE : CRISPR et drépanocytose : état de l'art et perspectives

📝 RÉSUMÉ EXÉCUTIF (3 phrases) :
"Des essais récents montrent que l'édition génique CRISPR-Cas9 
pourrait corriger la mutation HBB responsable de la drépanocytose. 
Les résultats précliniques sont prometteurs, mais la traduction 
clinique reste à valider."

🔑 POINTS CLÉS :
• La mutation HBB c.20A>T (p.Glu7Val) cause la drépanocytose
• CRISPR permet de cibler précisément cette mutation in vitro
• 3 essais cliniques de phase 1/2 sont en cours (PMID: XXXXX)
• Les défis : livraison in vivo, effets hors-cible, coût

📚 CITATIONS VÉRIFIABLES :
• "CRISPR editing of HBB in hematopoietic stem cells..." 
  → PMID: 12345678 | https://pubmed.ncbi.nlm.nih.gov/12345678
• "UniProt entry P68871: Hemoglobin subunit beta"
  → https://www.uniprot.org/uniprotkb/P68871

⚠️ LIMITES ET INCERTITUDES :
• La plupart des données sont précliniques (modèles murins)
• Peu d'informations sur la sécurité à long terme
• Accès limité aux données brutes des essais en cours

🎯 PROCHAINES ACTIONS CONCRÈTES :
• Contacter l'équipe du Dr. Smith (essai NCT01234567)
• Tester l'outil open-source "CRISPR-Analyzer" sur nos données
• Surveiller les publications des 3 prochains mois

🤖 AUTO-RÉFLEXION DE L'AGENT :
• J'ai appris que les variants HBB ont plusieurs noms (p.Glu7Val, E6V...)
• Je dois améliorer ma recherche sur les essais cliniques en cours
• Nouvelle hypothèse : croiser avec les données de pharmacovigilance

📊 VALEUR SCIENTIFIQUE : 0.85/1.0
✅ Utile maintenant : Oui — pistes actionnables identifiées
```

✨ **Chaque affirmation a une source.** Vous pouvez cliquer, vérifier, citer.

---

### 💾 Étape 5 : L'agent "range" et "partage" son travail

> *Comme vous qui enregistreriez un document et l'enverriez à vos collègues.*

- Il sauvegarde le rapport en **JSON** (pour réutilisation par d'autres outils)

📁 *Tout est organisé, daté, recherchaible.*

### 🧠 Étape 6 : L'agent "apprend" de son travail

> *Comme un chercheur qui note dans son carnet : "La prochaine fois, je ferai différemment."*

Tous les 5-10 cycles, l'agent s'auto-évalue :

```
🔍 Ce que j'ai bien fait :
• Mes requêtes PubMed ont trouvé 95% des articles pertinents
• J'ai correctement identifié les variants HBB

⚠️ Ce que je peux améliorer :
• J'ai raté 2 essais cliniques car je cherchais "trial" et pas "étude"

💡 Ma proposition d'amélioration :
• Modifier mon prompt pour inclure : "trial OR étude OR essai"

🧪 Comment je vais tester que ça marche :
• Relancer un cycle sur "TP53 cancer" et comparer le nombre de résultats
```

✅ **Puis il applique la modification tout seul** (avec sauvegarde de sécurité).

> 🤖 *L'agent devient plus intelligent à chaque cycle, sans que vous ayez à le reprogrammer.*

### Étapes d'installation

```bash
# 1. Cloner le repository
git clone https://github.com/LaurentVoanh/ai-adn-evo-2-mistral-Pollination-2026.git
cd ai-adn-evo-2-mistral-Pollination-2026

# 2. Créer les dossiers de stockage
mkdir -p data/{reports,memory,cache,proposals,auto_apis,insights}
mkdir -p logs
chmod 755 data logs

# 3. Copier le fichier d'environnement
cp .env.example .env

# 4. Éditer .env avec vos clés
nano .env
# → Ajouter vos clés Mistral API depuis https://console.mistral.ai/api-keys

# 5. Lancer l'installateur web
# Ouvrir : https://votre-domaine/install.php

# 6. Suivre l'assistant (2 minutes)
# → Vérifications système
# → Génération des fichiers
# → Configuration initiale

# 7. Supprimer les fichiers d'installation
rm install.php test.php

# 8. Accéder à l'interface
# Ouvrir : https://votre-domaine/index.php
```

### Installation sur Hostinger (shared hosting)


## 🧠 Auto-amélioration

### Comment ça marche ?

```
┌─────────────────────────────────────────────────────────────┐
│  CYCLE N                                                    │
│  → Action exécutée (ex: call_api PubMed)                    │
│  → Log JSONL : {"action":"call_api","success":true,...}     │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│  TOUS LES 5 CYCLES                                          │
│  → Lecture des 50 derniers logs JSONL                       │
│  → Calcul des métriques (succès, score, temps)              │
│  → Mistral analyse les patterns                             │
│  → Proposition d'amélioration (JSON structuré)              │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│  APPLICATION SÉCURISÉE                                      │
│  → Backup automatique du fichier cible                      │
│  → Syntax check PHP (php -l)                                │
│  → Test de validation (si défini)                           │
│  → Écriture ou rollback automatique                         │
│  → Log dans brain_history.jsonl                             │
└─────────────────────────────────────────────────────────────┘
```

### Types d'améliorations auto-générées

| Type | Cible | Exemple | Risque |
|------|-------|---------|--------|
| `prompt` | Prompts Mistral | Élargir termes PubMed | Low |
| `code` | Fonctions PHP | Optimiser extract_gene_symbols() | Low-Medium |
| `config` | Seuils | Ajuster min_relevance_score | Medium |
| `workflow` | Ordre des étapes | ClinVar avant PubMed | Medium |
| `discovery` | Nouvelles APIs | Ajouter GitHub API | Low |

### Consulter l'état du "cerveau"

```bash
# Voir l'état actuel
cat data/memory/brain_state.json
```

## 🔄 3. WORKFLOW COMPLET — DE GOOGLE NEWS AU RAPPORT PARFAIT

### Étape 1 : Google News (Déclencheur temporel)
```
→ L'agent scanne Google News RSS toutes les 3 minutes
→ Filtre par mots-clés scientifiques (gene, protein, CRISPR, cancer, etc.)
→ Exclut les sujets non-scientifiques (laptop, smartphone, prix, etc.)
→ Score de pertinence : 0.5 (base) + 0.07 par mot-clé scientifique détecté
→ Sélectionne le sujet avec le score le plus élevé (min 0.65)
```

### Étape 2 : Extraction automatique des entités (IA + Regex)
```
→ Mistral-tiny analyse le titre sélectionné
→ Extrait automatiquement :
   - Gènes (TP53, BRCA1, etc.)
   - Variants (R175H, p.Arg175His, etc.)
   - Maladies (cancer, Alzheimer, etc.)
   - Méthodes (CRISPR, RNA-seq, etc.)
→ Validation via Ensembl API (élimine les faux positifs)
→ AUCUNE SAISIE HUMAINE REQUISE
```

### Étape 3 : Génération des requêtes APIs (Mistral1)
```
→ Mistral-tiny génère 3-5 requêtes API précises
→ Utilise les entités extraites automatiquement
→ Diversifie les sources (PubMed + UniProt + Ensembl + ClinVar)
→ Format JSON strict, validé avant exécution
```

### Étape 4 : Appel des APIs scientifiques
```
→ Exécution parallèle des requêtes
→ Cache automatique (évite les appels redondants)
→ Retry automatique en cas d'échec (3 tentatives max)
→ Log de chaque appel dans logs/execution.jsonl
```

### Étape 5 : Synthèse et rapport (Mistral4)
```
→ Mistral-medium synthétise toutes les données brutes
→ Produit un rapport structuré :
   - Titre clair
   - Résumé exécutif (3-4 phrases)
   - Points clés (liste)
   - Citations vérifiables (PMID, UniProt ID, DOI)
   - Limites et incertitudes (obligatoire)
   - Prochaines actions concrètes
   - Auto-réflexion de l'agent
→ Score de valeur scientifique (0.0-1.0)
→ Validation rigueur (citations, hallucinations, méthodologie)
```

### Étape 6 : Publication et stockage
```
→ Sauvegarde JSON (data/reports/{slug}.json)
→ Génération HTML (data/reports/{slug}.html)
→ Indexation dans data/memory/index.json
→ Optionnel : Publication WordPress via REST API
```

## ⚠️ 6. DIFFICULTÉS PRATIQUES — HOSTINGER, COPY-PASTE, LIMITATIONS

### Problèmes connus et solutions

| Problème | Cause | Solution |
|----------|-------|----------|
| **Fichiers trop longs à copier-coller** | Limites navigateur/éditeur | Découper en modules de <500 lignes |
| **Hostinger : pas d'accès SSH** | Shared hosting | Utiliser File Manager ou FTP |
| **Hostinger : pas de cron** | Shared hosting | ✅ Notre solution est 100% AJAX, pas de cron requis |
| **Timeout PHP (30s par défaut)** | Configuration serveur | Ajouter `set_time_limit(0);` en tête de ajax.php |
| **Permissions d'écriture** | Dossier non writable | Créer manuellement `data/` et `logs/` via FTP, chmod 755 |
| **HTTPS obligatoire** | Sécurité navigateur | Forcer HTTPS dans .htaccess ou config Hostinger |
| **Taille max POST (2MB)** | Configuration PHP | Ajouter `php_value upload_max_filesize 10M` dans .htaccess |
| **Affichage des erreurs** | Production vs dev | `ini_set('display_errors', '0');` en prod, `'1'` en dev |
| **Clés API exposées** | Code dans le repo | Utiliser .env hors racine web + getenv() |

| **Rate limit Mistral** | Même clé utilisée | Rotation par index de cycle (`$keyIndex % 3`) |
